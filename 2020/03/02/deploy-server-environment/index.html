如何在Windows上搭建开发环境

发表于 2020-03-02 更新于 2020-03-09 阅读次数：

对于服务器开发人员来说，MacOS似乎是最理想的开发系统，但是大部分情况下，很多公司使用的还是Windows系统。
有两种解决方案，一是安装双系统或者单Ubuntu系统，这样电脑不会卡，保障了性能，又保障了开发效率，但是Unity等软件怎么运行是个问题，其次方案是装vmware虚拟机，这样日常开发就可以在虚拟机内的ubuntu上进行，内部的交流软件和Unity也可以在宿主机上运行，但是电脑的配置性能又成为瓶颈。在尝试多次后，我找到了貌似完美解决的方案，下面来聊一聊具体的实施步骤。

1.明确需求

首先要明确下服务器日常的办公需求：

  • 聊天软件 (内部软件 Only Windows)
  • Unity (Only Windows)
  • Intellij IDEA & GoLand (Windows or Ubuntu)
  • SVN & Git (Windows or Ubuntu)
  • 项目运行环境 (Only Ubuntu)

由此看来使用Windows作为主系统是一定的，但是虚拟机的方案对系统配置要求比较高，特别是又需要开Unity和IDEA，两个都是吃内存的主，但是又需要Ubuntu系统来运行服务器，所以想到的就是Dokcer，可以作为轻量级的虚拟机。
